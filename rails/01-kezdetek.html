Mi ez?

Nos kezdjük az elején, mi is ez? Az amit tudni lehet a Ruby On Rails ről, legalábbis az én ismereti szintemen, az az, hogy ez egy Ruby nyelvet használó WEBes keretrendszer :)

Mi a ruby? (https://hu.wikipedia.org/wiki/Ruby_(programoz%C3%A1si_nyelv))

A Ruby nyílt forráskódú, teljesen objektumorientált, interpretált, általános célú programozási nyelv.

Mi akkor a Rails?(RoR) (https://hu.wikipedia.org/wiki/Ruby_on_Rails)

A Ruby on Rails (röviden Rails) a Ruby programozási nyelvre épülő, nyílt forrású (MIT licenc alatti) webalkalmazás-keretrendszer.”

Lefordítva, Ruby nyelven fogod írni a weboldalad, amit a Rails fog neked a weben megjeleníteni. Kb ennyit bőven elég tudni róluk. :)

Hogyan kezdjek hozzá?

A ruby nyelv egy egészen egyszerű nyelv: http://nyelvek.inf.elte.hu/leirasok/Ruby/

Ror ról egy leírás, a saját honlapját érdemes bázisnak venni: (ijesztő, tudom, nyugi) https://guides.rubyonrails.org/

RoR telepítése windowsra:

  1. Töltsd le a Ruby nyelvet elsőnek : https://rubyinstaller.org/downloads/
  2. "WITH DEVKIT" válaszd ki, töltsd le a legújabbat, ez nekem jelenleg a “Ruby+Devkit 3.3.4-1 (x64)”
  3. Telepítsd :)

A telepítés befejezése után egy új ablak jelenik meg. Győződj meg róla, hogy a "Run 'ridk install' to setup MSYS2..." opció be van jelölve, majd kattints a Finish gombra.

RubyGems és Rails telepítése

Ruby telepítése után a RubyGems csomagkezelő automatikusan elérhetővé válik. Ezzel telepíthetjük a Rails-t.

a. Parancssor megnyitása

  • Nyisd meg a parancssort (CMD / Parancssor / Terminál) (Startmenübe írd be: CMD)
     
  • Ellenőrizd a Ruby telepítését a következő parancs beírásával:
     
ruby -v

Ez kiírja a telepített Ruby verzióját.

b. Rails telepítése

  • Írd be a következő parancsot a Rails telepítéséhez:
     
gem install rails

 

  • Várj, amíg a telepítés befejeződik.

c. Rails verzió ellenőrzése

Ellenőrizd a Rails telepítését a következő parancs beírásával:

rails -v

Ez kiírja a telepített Rails verzióját.

Kész is vagy :) 

Ror telepítése linuxra:

sudo apt install rails

 

Kész is vagy :)

Mi van a “motorháztető” alatt? 

Nos nézzük meg mit is telepített ez a csúf RAILS, miközben írogatta kifele a sorokat.

A RAILS MVC logikát követ, vagyis Modell, Nézet és Vezérlő.
Modell az ahol az adatbázisban való tevékenységek lesznek.

  • Adatbáziskapcsolat: lehetővé teszi az adatok lekérését, mentését, frissítését és törlését.
  • Validációk: Az adatok , lekérések ellenörzése
  • Asszociációk: Táblák közti kapcsolatok, viszonyok rögzítése

View (Nézet) A View felelős az adatok megjelenítéséért a felhasználó számára.

  • Sablonok: Olyan fájlok, amelyek dinamikusan generálják majd a HTML-t amit a felhasználó lát.
  • Partials: A nagyobb oldalak kisebb, újrahasználható részekre bonthatók, így könnyítve meg, hogy még egyszerűbben átlásd az adott oldal struktúráját.

Controller (Vezérlő) A Controller kezeli a felhasználói kéréseket, koordinálja a Model és a View közötti kommunikációt, és meghatározza, hogy milyen adatokat és nézeteket kell visszaadni a felhasználónak.

  • Akciók: A Controllerben az egyes akciók (pl. index, show, create, update, destroy) határozzák meg, hogy melyik nézetet kell megjeleníteni, és milyen adatokat kell lekérni vagy menteni.
  • Paraméterek kezelése: A Controller feldolgozza a bejövő HTTP kéréseket és kinyeri a szükséges paramétereket.
  • Irányítás: A Controller dönti el, hogy milyen műveletet kell végrehajtani, és milyen válasz szükséges (pl. HTML, JSON, stb.).

Talán ennyi elég is ebből :) A lényeg:

Megérkezik a kérés → CONTROLLER → Eldönti mit kell tennie → 

  • → Adatok kellenek? → MODELL
  • → Megjeleníteni kellene valamit?  VIEW 
  • → megjeleníti a HTML -t a felhasználónak, benne a MODEL ből lekért adatokkal

 

A könyvtárstruktúra WINDOWS és LINUX alatt kb megegyezik, néhány DLL -t kapsz WIN alá, meg máshogy néznek ki a fájlok, de összességében ugyanaz a felépítés.

APP könyvtár, itt vannak a WEBOLDAL fájljai.
CONFIG könyvtár, ahogy a neve elárúlja, itt lesznek a beállítások
Gemfile fájl, itt lesznek a használt GEM fájljaid! (Ezek a GEM ek egy olyan fájlok, programrészletek, amelyeket használva gyorsíthatsz a fejlesztéseden, mint máshol a “beépülő pluginok”)
 

APP könyvtáron belül :

  • assets/: Ez a könyvtár statikus fájlokat, például JavaScript, CSS és képfájlokat tartalmaz.
    • stylesheets/: CSS fájlok.
    • images/: Képfájlok.
  • controllers/: Itt találhatók a vezérlők, amelyek a HTTP kéréseket kezelik és az üzleti logikát végrehajtják.
  • helpers/: Segítő modulok találhatók itt, amelyek a nézetekben használt kiegészítő funkciókat tartalmazzák.
  • javascripts/: JavaScript fájlok.
  • models/: Itt helyezkednek el az alkalmazás adatmodelljei, amelyek az adatbázissal való kommunikációt kezelik.
  • views/: Ez a könyvtár tartalmazza az összes nézetet, amelyek a HTML sablonokat és más megjelenítési logikát tartalmazzák.
