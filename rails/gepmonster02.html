Ebben a cikkben létrehozzuk az első oldalunkat: 

Hozzuk létre a gep.monster alapjait az alábbi paranccsal CMD ben:

rails new monster

Lépjünk be a könyvtárba:

cd monster

A rails nagyon okosan nem bíz semmit a véletlenre, bár megteheted hogy kézzel hozol létre mindent, légy lusta :D

Az alábbi paranccsal létrehozzuk a GAMES oldalt, annak néhány paraméterével együtt.:

rails g scaffold game name:string description:text

Nézzük meg mit is kértünk a programunktól:

RAILS → ez a programunk, 

G → Generate , tehát készítsd el.

scaffold → Ez egy ügyes kis parancs, ami létrehozza az alapműveleteket az adott névhez.

name:string  → legyen egy name változó, ami string, tehát SZÖVEG alapú (a STRING maximum 255 karaktert tartalmazhat, elégnek kell lennie egy játék nevének)

description:text → legyen egy másik változója, ami ugyanúgy SZÖVEG alapú, de most TEXT, ami azt jelenti hogy soookal hosszabb lehet a beleírt szöveg (65,535 karakter hosszú lehet… ) 

Első körben ennyi elég kell hogy legyen egy játék leírásához :)
Kell még egy KÉP, amit manuálisan veszünk fel.
Lefuttatva a parancsot, ezt kapjuk:

Meg lehet figyelni, hogy a SCAFFOLD parancs milyen szépen létrehoz neked mindent, ami kellhet!

  • create    app/models/game.rb  → létrehozta a MODEL-t , tehát eléred az adatbázist.
  • route    resources :games  → beírta a ROUTE fájlba az elérését a weboldalnak → GAMES (Resources → minden létező elérét, a CREATE - EDIT - DELETE - SHOW részét is.)
  • create    app/controllers/games_controller.rb  → a CONTROLLER is létrejött, van ami irányítsa a kéréseket.
  • create      app/views/games   → Létrejött maga a weboldal része
    • create      app/views/games/index.html.erb →INDEX → minden itt fog megjelenni listázva
    • create      app/views/games/edit.html.erb→EDIT→ itt lehet szerkeszteni a játékot.
    • create      app/views/games/show.html.erb →SHOW→ Itt lehet a részleteit a játéknak megtekinteni.
    • create      app/views/games/new.html.erb →NEW→ Itt tudsz új játékot felvenni
    • create      app/views/games/_form.html.erb →FORM→ Ez a fő lelke a rendszernek, a FORMok 1 helyen vannak kigyűjtve (EDIT, NEW, így ha 1 helyen módosítod, mindenhol módosul…légy lusta )
    • create      app/views/games/_game.html.erb → GAME → Ezt hívja meg a rendszer a játékok megjelenítésére, elég 1 helyen módosítani, mindenhol egyszerre módosul…

Gratula, kész is a gep.monster “lelke” az index oldal :) 

Persze nézzük meg hogy is néz ki , ne várj tőle csodát, de azért ha a mélyére látsz, már látod hogy aham, ez az amit akarok!

indítsuk el a szervert :

rails s

majd hívjuk meg a weboldalt, az újat:

https://localhost:3000/games

Ha mindent jól csináltál eddig, egy szépséges ERROR fogad, mégpedig azért, mert nem a RAILS figyel rád :) Hogy ne kövess el hülyeséget.

Így felhívja a figyelmed arra, hogy talált egy olyan ADATBÁZIS migrációt, ami nincs felvéve a rendszerbe, “PENDING” alatt , azaz várakozás alatt van.

Van egy gomb: "Run pending migrations" → és felveszi a GAME adatbázisod (blabla_create_games.rb)
